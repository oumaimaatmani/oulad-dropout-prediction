# Early Dropout Detection with NLP -- Augmenting OULAD with Student Text Signals

## Overview

This project addresses early detection of student dropout in online higher education,
using the Open University Learning Analytics Dataset (OULAD). It combines classical
tabular machine learning with natural language processing to build a hybrid prediction
system that captures both quantitative engagement patterns and qualitative behavioral
signals.

The work progresses through five notebooks, each building on the previous:

1. **Data loading and exploration** -- raw OULAD ingestion, schema understanding, initial statistics
2. **Feature engineering** -- construction of behavioral, temporal, and assessment-based features
3. **Baseline modeling** -- XGBoost classifier on tabular features, evaluation, interpretation
4. **Advanced analysis** -- temporal patterns, cohort segmentation, SHAP explainability
5. **NLP-augmented prediction** -- DistilBERT sentiment classification, PyTorch MLP hybrid model

## Dataset

The Open University Learning Analytics Dataset contains anonymized data from
courses presented at the Open University (UK) between 2013 and 2014. It includes:

- **studentInfo**: demographics, registration, and final outcome
- **studentRegistration**: registration and unregistration dates
- **studentAssessment**: individual assessment scores
- **studentVle**: daily interaction counts with virtual learning environment resources
- **assessments**: assessment metadata (type, weight, due date)
- **vle**: VLE resource metadata (activity type, week)
- **courses**: module presentation metadata

Source: https://analyse.kmi.open.ac.uk/open_dataset

The dataset must be downloaded separately and placed in `data/raw/` before
running the notebooks. See the Setup section below.

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/your-username/oulad-dropout-prediction.git
cd oulad-dropout-prediction
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# venv\Scripts\activate   # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download the OULAD dataset

Download from https://analyse.kmi.open.ac.uk/open_dataset and extract all
CSV files into `data/raw/`:

```
data/raw/studentInfo.csv
data/raw/studentRegistration.csv
data/raw/studentAssessment.csv
data/raw/studentVle.csv
data/raw/assessments.csv
data/raw/vle.csv
data/raw/courses.csv
```

### 5. Run the notebooks in order

```bash
jupyter notebook notebooks/
```

Execute notebooks 01 through 05 sequentially. Each notebook saves its outputs
to `data/processed/` and `results/`.

## Methodology

### Phase 1 -- Tabular Baseline (Notebooks 1-3)

The baseline system extracts behavioral features from VLE interaction logs and
assessment records, then trains an XGBoost gradient boosting classifier.

Key features include:
- Cumulative VLE clicks and distinct activity counts
- Assessment completion ratio and weighted average scores
- Learning pace (deviation from recommended assessment schedule)
- Module engagement rate and activity diversity index
- Registration timing relative to course start
- Temporal engagement trends (weekly slopes)

The XGBoost model is trained with stratified 5-fold cross-validation and
evaluated on ROC-AUC, precision, recall, and F1-score.

### Phase 2 -- Advanced Analysis (Notebook 4)

This phase adds:
- SHAP (SHapley Additive exPlanations) feature importance analysis
- Temporal engagement pattern clustering
- Cohort-level segmentation by demographics and prior education
- Early warning threshold calibration

### Phase 3 -- NLP Augmentation (Notebook 5)

Building on the XGBoost baseline, this extension integrates:
- **DistilBERT** (HuggingFace Transformers) for sentiment classification
  of student behavioral profiles expressed as natural language descriptions
- **PyTorch MLP** combining NLP-derived features with tabular engagement signals
- Forum engagement analysis as a proxy for social learning disengagement

This moves the system from tabular ML toward a hybrid neuro-symbolic architecture
better suited for real-world deployment in LMS platforms, where student-generated
text (forum posts, messages, self-assessments) provides rich predictive signal
beyond click-level telemetry.

The hybrid model architecture:
1. Converts each student's feature vector into a natural language description
2. Passes descriptions through DistilBERT to extract sentiment and confidence
3. Combines NLP outputs with tabular features in a feedforward neural network
4. Trains end-to-end with binary cross-entropy loss

## Results Summary

| Model                      | ROC-AUC | Precision | Recall | F1    |
|----------------------------|---------|-----------|--------|-------|
| XGBoost (tabular only)     | 0.91+   | 0.84+     | 0.79+  | 0.81+ |
| MLP + NLP features         | 0.92+   | 0.85+     | 0.81+  | 0.83+ |

Results are approximate and depend on the specific train/test split.
Full classification reports are generated in notebooks 3 and 5.

## Reproducibility

All notebooks set explicit random seeds:
- `numpy.random.seed(42)`
- `torch.manual_seed(42)`
- XGBoost `random_state=42`

Data splits use `StratifiedKFold` with fixed `random_state` to ensure
identical folds across runs.
